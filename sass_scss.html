SASS & SCSS

A SASS és az SCSS egyaránt a SASS (Syntactically Awesome Stylesheets) előfeldolgozó szintaxisa, amelyek mindegyike olyan továbbfejlesztett funkciókat kínál, amelyek kiterjesztik a hagyományos CSS képességeit. Bár mindkettő ugyanazokat a funkciókat kínálja, szintaxisuk eltérő, így a fejlesztők eltérő preferenciáit is kielégíti.

SCSS

Az SCSS (Sassy CSS) a SASS egy szintaxisa, amely CSS-szerű struktúrát alkalmaz. Ez megkönnyíti a CSS-sel már ismerkedő fejlesztők számára az átállást a SASS által kínált haladó funkciók, például a változók, a mixinek és a beágyazás használatára. Az SCSS fájlok kiterjesztése .scss, és teljes mértékben kompatibilisek a szabványos CSS-sel, ami azt jelenti, hogy bármilyen érvényes CSS-t használhatsz egy SCSS fájlban.

  • Változók: Az egységes stílus érdekében tároljon újrafelhasználható értékeket, például színeket és betűtípusokat.
  • Beágyazás: A CSS szelektorok hierarchikus módon ágyazódnak be a jobb olvashatóság érdekében.
  • Mixins: Újrafelhasználható stílusblokkok létrehozása az ismétlődő kód elkerülésével.
  • Öröklődés: Az @extend használatával megoszthatja a stílusokat a szelektorok között, ami leegyszerűsíti a kódbázist.
  • Részleges stílusok és importálás: Moduláris CSS az @import használatával, így a stílusok rendezettek és karbantarthatók maradnak.

SASS

A Sass a legnépszerűbb és legrégebbi CSS előfeldolgozó, amely eredetileg 2006-ban jelent meg. Alkotóit, Natalie Weizenbaumot és Hampton Catlint a Haml sablonnyelv ihlette, amely dinamikus funkciókat ad a HTML-hez. Céljuk az volt, hogy hasonló dinamikus funkciókat valósítsanak meg a CSS-ben is. Így hát létrehoztak egy CSS előfeldolgozót, és Syntactically Awesome Style Sheets-nek nevezték el.

A Sass előfeldolgozó lehetővé teszi számunkra változók, if/else utasítások, for/while/each ciklusok, öröklődés, operátorok, interpoláció, mixin-ek és egyéb dinamikus funkciók használatát, majd a kódot egyszerű CSS-sé fordítjuk, amelyet a webböngészők értelmezni tudnak.

Előnyök:

  • jól bevált, stabil CSS előfeldolgozó
  • A fordítás könnyebbé vált a Dart Sass bevezetésével
  • nagy ökoszisztémával rendelkezik
  • fejlett dinamikus funkciók
  • két szintaxis közül választhatsz (Sass és SCSS - lásd az alábbi példákat)
  • kiterjedt dokumentáció

LESS

A LESS-t három évvel a Sass után, 2009-ben adta ki Alexis Sellier. A Sass befolyásolta, így számos funkcióját megvalósítja, mint például a mixineket, a változókat és a beágyazást. Vicces módon később a LESS is hatással volt a Sass-re, mivel az újabb SCSS szintaxist a LESS szintaxisa ihlette.

A LESS CSS előfeldolgozó egy JavaScript könyvtár, amely kibővíti a CSS alapértelmezett funkcióit. Mivel JavaScriptben van megírva, futtathatod npm csomagként , vagy lefordíthatod a LESS-t közvetlenül a böngészőben .lessfájlok és a LESS konverter hozzáadásával a HTML oldalad szakaszához.

Előnyök:

  • közvetlenül hozzáadható egy HTML oldalhoz
  • lapos tanulási görbe, mivel a LESS a standard CSS szintaxist használja (lásd az alábbi példákat)
  • számos dinamikus funkció, beleértve a hatókörbe tartozó változókat, importálási opciókat, mixineket, beágyazott szelektorokat és egyebeket
  • bővítményeken keresztül bővíthető
  • részletes dokumentáció
